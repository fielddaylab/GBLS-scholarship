## Abstract

The reference transaction can be understood as a system and modeled as such. By adopting a problem-solving paradigm other than the one traditionally used in this area (i.e., flowcharting), a systems analysis of the question-answering process can yield new insights. In particular, this article identifies fifteen functional requirements of the process as well as fit criteria to measure success in question negotiation. In addition, the process is graphically illustrated using four diagrams of the reference transaction and provides a top-level perspective of the process.
